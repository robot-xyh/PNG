# 穿越机拦截无人机系统 - 总体方案

## 一、系统概述

### 1.1 任务目标
使用穿越机自主拦截来袭无人机，实现高精度侧向拦截。

**第一阶段性能指标：**
- 拦截器速度：50 m/s
- 目标速度：20 m/s（匀速直线运动）
- 拦截距离：50-500m
- **脱靶量要求：< 0.5m**
- 拦截场景：侧向拦截（初始视线角约90°）

## 二、核心技术方案

### 2.1 坐标系统

**三个坐标系：**

1. **惯性坐标系 (I) - NED**
   - Xi: 北，Yi: 东，Zi: 下

2. **机体坐标系 (B)**
   - Xb: 前，Yb: 右，Zb: 下

3. **相机坐标系 (C) - 捷联安装**
   - Xc: 右，Yc: 下，Zc: 光轴（斜向上45°）

**关键变换：**
```
相机→机体：R_BC = Ry(45°)
机体→惯性：R_IB = f(四元数姿态)
相机→惯性：R_IC = R_IB · R_BC
```

### 2.2 惯导对准（系统启动关键环节）

#### 阶段1：粗对准（静止30秒）
- **输入**：加速度计（重力）+ 磁力计（地磁场）
- **输出**：初始姿态（横滚、俯仰、航向）
- **精度**：横滚/俯仰 ±0.5°，航向 ±2°

#### 阶段2：精对准（飞行中，GPS辅助）
- **条件**：低速飞行（>5m/s）
- **方法**：GPS速度矢量与机体速度对比修正航向
- **持续时间**：飞行前5分钟持续修正

**重要性：** 对准误差直接导致坐标变换错误，影响导引精度

---

### 2.3 视觉检测与跟踪

#### YOLO目标检测
- **模型**：YOLOv8n（轻量级）
- **部署**：RKNN格式，RK3588 NPU加速
- **性能**：<30ms推理时间，60fps实时检测
- **输出**：[x, y, w, h, confidence, class]

#### ByteTrack多目标跟踪
- **功能**：关联前后帧检测，稳定跟踪ID
- **优势**：处理遮挡、短暂丢失
- **输出**：track_id, bbox, confidence

#### 目标选择策略
- 优先级：距离最近 + 置信度最高
- 锁定后持续跟踪同一ID

---

### 2.4 视线角速度估计（核心算法）

#### 卡尔曼滤波器设计

**状态向量（8维）：**
```
X = [λx, λy, λz, λ̇x, λ̇y, λ̇z, r, ṙ]ᵀ

λ  : 视线单位向量（惯性系）
λ̇  : 视线向量时间导数
r  : 目标距离
ṙ  : 距离变化率
```

**观测量（3维）：**
```
Z = [u, v, bbox_size]ᵀ

u, v       : 目标像素坐标
bbox_size  : 边界框尺寸（用于估计距离）
```

**滤波流程：**
```
1. 预测步：X_{k+1|k} = F·X_k + w
2. 观测步：从像素(u,v)反推视线方向
3. 更新步：融合观测，修正状态
4. 输出：λ̇ → 计算视线角速度 ω_los = λ × λ̇
```

**关键点：**
- 距离估计：通过已知目标尺寸和边界框大小
- 噪声调参：过程噪声Q vs 观测噪声R平衡
- 归一化：每步后重新归一化λ保持单位向量

---

### 2.5 比例导引律（PNG）

#### 原理
```
a_cmd = N · V_c · ω_los

N     : 导引比（本系统=4.5，侧向拦截优化）
V_c   : 接近速度（closing velocity）
ω_los : 视线角速度
```

#### 物理意义
- 加速度方向：垂直于视线
- 目标：保持视线角速度→0，距离→0
- 结果：追踪碰撞三角形，实现拦截

#### 侧向拦截特点
```
初始状态：
- 拦截器速度：50 m/s
- 目标速度：20 m/s
- 初始视线角：~90°（垂直）
- 初始距离：300-500m

导引过程：
1. PNG产生垂直于视线的横向加速度
2. 拦截器轨迹弯曲，"追赶"目标
3. 视线角速度逐渐减小
4. 碰撞点：ω_los→0, r→0
```

#### 优化措施
- **低通滤波**：ω_los滤波（α=0.3）去除高频噪声
- **加速度限制**：30 m/s²（约3G，第一阶段保守值）
- **垂直化处理**：确保加速度垂直于视线

### 2.6 控制架构

#### 控制循环（50Hz）

```
┌──────────────────────────────────────┐
│  主控制循环 (20ms周期)                │
├──────────────────────────────────────┤
│                                      │
│  1. 获取飞机状态 (3ms)                │
│     - MAVLink: 位置、速度、姿态       │
│                                      │
│  2. 图像处理 (异步，60Hz线程)         │
│     - YOLO检测 + ByteTrack跟踪       │
│                                      │
│  3. 卡尔曼滤波 (5ms)                  │
│     - 预测 + 更新                    │
│     - 输出：ω_los, λ, r              │
│                                      │
│  4. 比例导引 (2ms)                    │
│     - 计算加速度指令                 │
│     - 积分为速度指令                 │
│                                      │
│  5. MAVLink发送 (2ms)                │
│     - 速度控制指令                   │
│                                      │
│  总延迟：< 12ms（留8ms余量）          │
└──────────────────────────────────────┘
```

#### 多线程设计
```python
Thread 1: 相机采集（60 Hz）
Thread 2: YOLO检测（60 Hz，NPU）
Thread 3: 主控制（50 Hz）- 卡尔曼+PNG+MAVLink
```

#### 状态机
```
INIT → ALIGNED → SEARCH → TRACK → INTERCEPT
  ↑                          ↓
  └──────── 目标丢失 ─────────┘
```

## 三、系统工作流程

### 3.1 启动流程

```
1. 系统上电
   ├─ RK3588启动
   ├─ 连接Pixhawk（MAVLink）
   └─ 相机初始化

2. 惯导对准
   ├─ 粗对准（静止30秒）
   │  └─ 提示：请保持飞机静止
   └─ 精对准标志（飞行中持续）

3. 系统就绪
   └─ 等待起飞和拦截指令
```

### 3.2 拦截流程

```
1. 搜索阶段
   ├─ 低速盘旋或指定航线
   └─ 实时检测目标

2. 锁定目标
   ├─ YOLO检测到目标
   ├─ ByteTrack分配track_id
   └─ 状态切换：SEARCH → TRACK

3. 跟踪导引
   ├─ 60Hz检测跟踪
   ├─ 50Hz卡尔曼滤波
   ├─ 50Hz PNG计算
   └─ 50Hz速度指令

4. 拦截判定
   ├─ r < 0.5m → 拦截成功
   ├─ r > 5m 且 V_c > 0 → 脱靶
   └─ 否则 → 继续导引
```

### 3.3 异常处理

```
- 目标丢失：切换回SEARCH，重新搜索
- MAVLink超时：紧急悬停
- 相机故障：安全降落
- 控制超时：日志警告，继续运行
```
